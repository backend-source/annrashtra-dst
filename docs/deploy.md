# Go-live deployment guide (pilot)

Pilot setup: **API on Render (free)** + **dashboard on Cloudflare Pages (free)** +
**database on Neon** (already created). OTP is off for now, so promoters log in by
mobile. MSG91/WhatsApp and the outbox worker come later.

Order matters: **GitHub → Render (API) → Cloudflare (dashboard) → CORS**. Each step
needs a URL from the previous one.

> Use the **business email** for GitHub, Render, and Cloudflare so the company owns
> everything (see [credentials-setup.md](credentials-setup.md)).

---

## Step 1 — GitHub (host the code)

1. Create an account at https://github.com using the business email; verify the email.
2. Create a new **private** repository named `annrashtra-dst` (no README/.gitignore —
   the repo already has them).
3. Push this local repo to it. From `C:\Users\Dell\annrashtra-dst`:
   ```bash
   git remote add origin https://github.com/<your-username>/annrashtra-dst.git
   git push -u origin master
   ```
   GitHub will ask you to sign in (a browser window or a Personal Access Token).

What does NOT get pushed (gitignored, kept secret): `api/.env`, `google-services.json`,
keystores. Only source code goes up.

---

## Step 2 — Render (the API)

1. Sign up at https://render.com with the business email → "Sign in with GitHub" and
   authorize Render to read the `annrashtra-dst` repo.
2. **New + → Blueprint** → select the repo. Render reads [`render.yaml`](../render.yaml)
   and proposes the `annrashtra-dst-api` web service (Singapore, free).
3. Before the first deploy it asks for the two secret env vars:
   - `DATABASE_URL` → paste the **Neon connection string** (the one ending
     `...neon.tech/neondb?sslmode=require`).
   - `CORS_ORIGIN` → leave blank for now; we set it in Step 4.
   (`JWT_SECRET` is auto-generated by Render; `NODE_ENV`, `OTP_DISABLED` are preset.)
4. Click **Apply / Deploy**. When it's live you get a URL like
   `https://annrashtra-dst-api.onrender.com`.
5. Test it: open `https://annrashtra-dst-api.onrender.com/health` → should show
   `{"ok":true}`. **Copy this API URL** — both the dashboard and the APK need it.

> Free plan note: after ~15 min idle the API sleeps; the next request takes ~50s to
> wake. Normal for a pilot. Upgrade to the $7/mo plan later for always-on.

### One-time: run the database migrations against the prod DB
The Neon DB already has the schema from development. If you ever point at a fresh DB,
run from `api/` locally with that `DATABASE_URL` in `.env`:
```bash
npm run migrate
node scripts/db-migrate.js ../db/schema_v2.sql ../db/schema_v3.sql ../db/schema_v4.sql ../db/schema_v5.sql ../db/schema_v6.sql
```

---

## Step 3 — Cloudflare Pages (the dashboard)

1. Sign up at https://dash.cloudflare.com with the business email.
2. **Workers & Pages → Create → Pages → Connect to Git** → authorize and pick
   `annrashtra-dst`.
3. Build settings:
   - **Production branch:** `master`
   - **Root directory:** `dashboard`
   - **Build command:** `npm install && npm run build`
   - **Build output directory:** `dist`
4. **Environment variable** (Build): `VITE_API_BASE` = the Render API URL from Step 2
   (e.g. `https://annrashtra-dst-api.onrender.com`). This is baked in at build time.
5. **Save and Deploy.** You get a URL like `https://annrashtra-dst.pages.dev`.
   **Copy this dashboard URL.**

---

## Step 4 — Connect them (CORS) + verify

1. Back in **Render → annrashtra-dst-api → Environment**, set `CORS_ORIGIN` to the
   Cloudflare URL (e.g. `https://annrashtra-dst.pages.dev`). Save → Render redeploys.
2. Open the dashboard URL, log in with the **admin mobile** (no OTP) → you should see
   the Overview. If login fails with a CORS/network error, re-check `CORS_ORIGIN`
   exactly matches the dashboard URL (no trailing slash).

---

## Step 5 — Build the APK pointing at the live API

From `mobile/`:
```bash
flutter build apk --release --dart-define=API_BASE=https://annrashtra-dst-api.onrender.com
```
Output: `mobile/build/app/outputs/flutter-apk/app-release.apk` — share this with the
promoters. (For Play Store you also need release signing — see credentials-setup.md.)

---

## Later (when MSG91/DLT + WhatsApp are live)
- Add the `MSG91_*` env vars on Render.
- Add a Render **worker** service running `node scripts/outbox-worker.js` (paid plan)
  to send queued OTP/WhatsApp messages.
- Set `OTP_DISABLED=false` on Render to switch login back to OTP.
- No code change or APK rebuild needed for the OTP switch.
